# Architecture Baseline

## Decision

Use the final conceptual architecture from the start, and reduce only the active implementation profile for the hackathon.

## Rationale

This avoids a throwaway prototype and keeps the hackathon build on the same path as the intended product.

## Consequences

- core types must be designed now;
- transport abstraction must be real now;
- package and node separation must be real now;
- some surfaces may stay intentionally narrow in the hackathon build.
