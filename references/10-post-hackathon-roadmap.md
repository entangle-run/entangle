# Post-Hackathon Roadmap

## Phase 1: harden the architecture

- stabilize package schema
- stabilize graph schema
- stabilize A2A protocol
- improve runner reliability
- improve memory update discipline
- add validation tooling

## Phase 2: improve the client

- richer graph editor
- stronger topology diffing
- better runtime playback
- better node configuration UX
- graph templates for common organizations

## Phase 3: remote nodes and federation

- attach remote nodes by identity and metadata
- support richer transport policies
- support multi-relay profiles
- support better artifact fetch/push coordination

## Phase 4: ecosystem

- public SDK
- package validation and publishing tools
- node compatibility testing
- curated package registries

## Phase 5: economics and governance

- pricing and settlement
- local trust scoring
- policy packs
- approval workflows
- enterprise-grade governance and audit

## Long-term target

The long-term target is not just a better agent shell. It is an operating environment for AI organizations built on portable packages, graph-structured governance, signed protocol transport, and durable shared artifacts.
