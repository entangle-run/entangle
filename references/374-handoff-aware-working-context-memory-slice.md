# Handoff-Aware Working Context Memory Slice

## Current Repo Truth

Runner turns already persist `emittedHandoffMessageIds`, and autonomous
handoff messages are part of the runner-owned coordination model. Model-guided
memory synthesis already receives session, artifact, execution, and
source-change evidence, and `working-context.md` carries deterministic
source-change context even when the model prose omits it.

Before this slice, emitted handoff ids were not rendered into the memory
synthesis prompt or the durable working-context summary.

## Target Model

Each node wiki should retain bounded coordination evidence when a turn delegates
work to another graph node. Handoff memory must stay compact and deterministic:
store message ids, not full peer logs or duplicated conversation transcripts.

## Concrete Changes Required

- Add bounded handoff evidence to the model-guided memory synthesis prompt.
- Add a deterministic `Handoff Context` section to `working-context.md`.
- Keep the section sourced from `RunnerTurnRecord.emittedHandoffMessageIds`
  instead of relying on model-generated prose.
- Extend memory-synthesizer coverage to assert prompt and wiki output.

## Migration/Compatibility Notes

No schema change is required. Existing turn records already default
`emittedHandoffMessageIds` to an empty list, and old working-context pages can
be regenerated by a later synthesis pass.

## Risks And Mitigations

- Risk: memory pages grow with every delegated conversation.
  Mitigation: the section stores only bounded handoff message ids already
  present on the turn record.
- Risk: the model hallucinates handoff state.
  Mitigation: the durable section is rendered from typed runner state, not from
  the model's natural-language output.
