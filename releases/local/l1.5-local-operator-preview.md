# L1.5 Local Operator Preview

Status: released.

Release date: 2026-04-25.

Tag: `v0.1.5-local-operator-preview`.

Product line: Entangle Local.

Production claim: none.

## Release Summary

L1.5 makes Entangle Local easier to try without requiring the operator to read
the runtime smoke implementation first. It adds canonical Local Preview assets
and a near-one-command demo path that uses the same local host, runner, relay,
model-stub, and Gitea/git-backed artifact flow as the runtime smoke.

Included:

- canonical Local Preview package assets under `examples/local-preview/`;
- canonical Local Preview graph and model-stub catalog examples;
- `pnpm ops:demo-local-preview` as the near-one-command local demo path;
- `pnpm ops:demo-local-preview:reset` as the documented reset path for demo
  state;
- Local Preview demo mode in the existing runtime smoke path, reusing the same
  host, runner, local relay, model-stub, and Gitea/git-backed artifact flow;
- CLI and Studio inspection steps after the demo completes;
- troubleshooting notes for the common local failure modes.

Excluded:

- Local GA;
- Cloud or Enterprise implementation;
- production persistence, authentication, authorization, scheduling, sandboxing,
  or compliance claims;
- polished first-run onboarding;
- Studio session launch;
- graph template management;
- artifact preview/history workbench;
- productized backup, restore, repair, or upgrade tooling.

## Demo Path

Run:

```bash
pnpm install --frozen-lockfile
pnpm ops:demo-local-preview
```

The command starts the Local Compose profile, verifies Studio, host, Gitea, and
`strfry`, runs the Local Preview runtime path, and leaves the profile running
for inspection.

Inspect from the CLI:

```bash
pnpm --filter @entangle/cli dev host status --summary
pnpm --filter @entangle/cli dev host graph get --summary
pnpm --filter @entangle/cli dev host sessions list --summary
pnpm --filter @entangle/cli dev host runtimes artifacts local-preview-planner --summary
pnpm --filter @entangle/cli dev host runtimes artifacts local-preview-builder --summary
pnpm --filter @entangle/cli dev host events list --runtime-trace-only --summary
```

Inspect Studio at `http://localhost:3000`.

Reset:

```bash
pnpm ops:demo-local-preview:reset
```

## Troubleshooting

- Docker daemon unavailable: run `pnpm ops:check-local:strict` and fix Docker
  daemon access before starting the preview.
- Port collision: stop the process using the default Local ports or set the
  `ENTANGLE_STUDIO_PORT`, `ENTANGLE_HOST_PORT`, `ENTANGLE_GITEA_HTTP_PORT`,
  `ENTANGLE_GITEA_SSH_PORT`, or `ENTANGLE_STRFRY_PORT` environment variables.
- Stale Gitea or host state: run `pnpm ops:demo-local-preview:reset` before a
  fresh demo attempt.
- Missing model secret: the preview command writes the local model-stub secret
  into the host secret volume; rerun the command after reset if a prior attempt
  was interrupted before secret injection.
- Relay unreachable: confirm `pnpm ops:smoke-local` passes and that the
  `strfry` service is running on the configured relay port.
- Runner degraded: inspect
  `pnpm --filter @entangle/cli dev host runtimes list --summary`, then check
  `host events list --runtime-trace-only --summary` and rerun after reset if
  stale state is involved.

## Known Limitations

- The preview demo uses a local OpenAI-compatible model stub, not a real model
  provider.
- The graph includes review and handoff topology, but the runnable task path
  records an approval-bypass state rather than a polished approval workflow.
- The demo command leaves the Local profile running for inspection and requires
  an explicit reset command.
- The Local runtime profile still carries the legacy machine value
  `hackathon_local`.
- This packet is not a GA release packet and must not be used as a production
  readiness claim.
