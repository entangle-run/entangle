# Entangle Documentation

This directory is the operator-facing documentation for the Entangle runtime.
It is intentionally much smaller than `references/`.

Use these files first:

- [architecture.md](architecture.md) - the current runtime architecture.
- [running-entangle.md](running-entangle.md) - how to boot and prove the system.
- [testing.md](testing.md) - quality gates and smoke-test selection.
- [status.md](status.md) - implemented, deterministic, manual, and remaining
  capability areas.
- [repository-guide.md](repository-guide.md) - repository organization and
  maintenance rules.
- [repository-and-website-redesign.md](repository-and-website-redesign.md) -
  the documentation and website cleanup plan that created this docs surface.

Use `references/` when you need design history, implementation ledgers, or
slice-level detail. Use `wiki/` when you need long-running project memory.
